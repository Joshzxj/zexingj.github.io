Why Must Debugging Be Done by Yourself

Posted on Jan 18, 2024

If someone points out where your program is wrong, you won’t gain any insight, and you may repeat the same mistake next time. This is because the errors you discover through extensive debugging leave a deeper impression. Debugging ability is a key measure of a coder’s productivity and can enhance your capacity to write high-quality, bug-free code.

This leads me to think about the steps of debugging—not just to fix the issue, but also to push myself to delve deeper. Here are my four steps:

  1. Reread the program. Follow the thought process you had when writing it and check if the program is executing as you intended. Use print statements to log intermediate results. I call this “debugging by observation without compiling.”
  2. Find a very small input that can cause your program to fail. This means you are “pretending” to run the program with actual input while reasoning about the expected output.
  3. If you encounter any unexpected behavior, narrow down the problem and ensure it remains within your scope.

What To Do If You Really Can’t Debug?

If you have been debugging for an entire day and still can’t find a solution, it’s okay to ask someone else for help. However, make sure you have thoroughly explored the problem yourself first. Don’t rush to seek assistance without thinking deeply about the issue. Check Ask Question Wisely
